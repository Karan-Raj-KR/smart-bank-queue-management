# Smart Bank Queue Management

![Python](https://img.shields.io/badge/Python-3.11%2B-3776AB?style=flat&logo=python&logoColor=white)
![FastAPI](https://img.shields.io/badge/FastAPI-0.100%2B-009688?style=flat&logo=fastapi&logoColor=white)
![PostgreSQL](https://img.shields.io/badge/PostgreSQL-15%2B-336791?style=flat&logo=postgresql&logoColor=white)

A backend API for managing bank branch queues — handling token issuance, priority routing, real-time status updates, and ML-powered wait time predictions.

---

## Features

- **Token Generation** — Issue numbered queue tokens with priority levels tied to service type
- **Priority Queue** — Counters call the next customer using a priority-based ordering strategy
- **Real-time Updates via WebSockets** — Clients subscribe to live queue state per branch
- **SMS Notifications** — Twilio integration sends SMS alerts when a customer's token is called
- **ML Wait Time Prediction** — scikit-learn linear regression model predicts wait time from queue length, hour, day, and service type
- **Analytics Dashboard** — Per-branch endpoints for daily summary, hourly breakdown, and service-type breakdown

## System Architecture

```
smart-bank-queue-management/
├── app/
│   ├── core/           # Database session, settings (env vars via pydantic-settings)
│   ├── models/         # SQLAlchemy ORM models (Token, Counter, Branch, Customer, ServiceType)
│   ├── schemas/        # Pydantic request/response schemas
│   ├── routers/        # FastAPI route handlers (tokens, counters, websocket, analytics)
│   ├── services/       # Business logic (token generation, counter service)
│   ├── tasks/          # Background tasks (Twilio SMS notifications)
│   └── main.py         # App entry point, router registration
├── ml/
│   ├── train.py        # Trains and serializes the wait time prediction model
│   └── model.pkl       # Serialized LinearRegression model (generated by train.py)
├── alembic/            # Database migration scripts
├── tests/              # Test suite
└── docker-compose.yml  # PostgreSQL + Redis services
```

## Getting Started

### Prerequisites

- Docker & Docker Compose
- Python 3.11+

### Setup

```bash
# 1. Clone the repository
git clone https://github.com/your-username/smart-bank-queue-management.git
cd smart-bank-queue-management

# 2. Configure environment variables
cp .env.example .env
# Edit .env with your database, Redis, and Twilio credentials

# 3. Start infrastructure (PostgreSQL + Redis)
docker-compose up -d

# 4. Install Python dependencies
pip install -r requirements.txt

# 5. Run database migrations
alembic upgrade head

# 6. Train the ML model
python ml/train.py

# 7. Start the development server
uvicorn app.main:app --reload
```

The API will be available at `http://localhost:8000`.
Interactive docs: `http://localhost:8000/docs`

## API Endpoints

| Method | Path                                       | Description                                      |
|--------|--------------------------------------------|--------------------------------------------------|
| GET    | `/`                                        | Root                                             |
| GET    | `/health`                                  | Service health status                            |
| POST   | `/tokens/generate`                         | Issue a new queue token for a customer           |
| POST   | `/counters/call-next`                      | Call the next token at a counter                 |
| GET    | `/counters/queue/{branch_id}`              | Get current queue status for a branch            |
| WS     | `/ws/{branch_id}`                          | Subscribe to live queue updates                  |
| GET    | `/analytics/{branch_id}/summary`           | Daily summary (total tokens, avg wait, no-shows) |
| GET    | `/analytics/{branch_id}/hourly`            | Hourly token count and average wait time         |
| GET    | `/analytics/{branch_id}/service-breakdown` | Token count and wait time by service type        |

## Environment Variables

Create a `.env` file in the project root based on `.env.example`:

| Variable               | Description                                | Example                                                           |
|------------------------|--------------------------------------------|-------------------------------------------------------------------|
| `DATABASE_URL`         | Async PostgreSQL connection string         | `postgresql+asyncpg://postgres:postgres@localhost:5432/bankqueue` |
| `REDIS_URL`            | Redis connection string                    | `redis://localhost:6379`                                          |
| `TWILIO_ACCOUNT_SID`   | Twilio account SID                         | `ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`                              |
| `TWILIO_AUTH_TOKEN`    | Twilio auth token                          | `your_auth_token`                                                 |
| `TWILIO_PHONE_NUMBER`  | Twilio sender phone number (E.164 format)  | `+1234567890`                                                     |
| `DEBUG`                | Enable debug mode                          | `false`                                                           |
